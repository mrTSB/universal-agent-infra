Aeon has two parts: a durable agent server, and a Python SDK that talks to it.

## Prerequisites

<Columns cols={3}>
  <Card title="Python 3.9+" icon="code">
    Required for the `aeon-sdk` package.
  </Card>

  <Card title="Node, pnpm, or Bun" icon="terminal">
    Required to run the local Aeon server.
  </Card>

  <Card title="Anthropic API key" icon="key">
    Required for model-backed agent runs.
  </Card>
</Columns>

## Set up Aeon

<Steps>
  <Step title="Install the Python SDK">
    ```bash
    python -m pip install aeon-sdk
    ```

    The package name is `aeon-sdk`; you import it as `aeon`:

    ```python
    from aeon import Agent, Objective, Runtime
    ```
  </Step>

  <Step title="Configure credentials">
    ```bash
    pnpm install
    cp .env.example .env
    # set ANTHROPIC_API_KEY=sk-ant-...
    ```
  </Step>

  <Step title="Start the server">
    The SDK requires a running Aeon server.

    <Tabs>
      <Tab title="pnpm">
        ```bash
        pnpm run agent
        ```
      </Tab>

      <Tab title="Bun">
        ```bash
        bun run agent
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Point the SDK at another server">
    By default the SDK connects to `http://localhost:3000`. Override it from the shell:

    ```bash
    export AEON_SERVER=http://your-host:3000
    ```

    Or pass a URL to `Runtime`:

    ```python
    from aeon import Runtime

    runtime = Runtime("http://remote:3000")
    ```
  </Step>
</Steps>

## Environment variables

<Columns cols={2}>
  <Card title="ANTHROPIC_API_KEY" icon="key">
    Required for model runs. Direct Anthropic API key for the main agent.
  </Card>

  <Card title="AEON_SERVER" icon="server">
    Optional SDK server URL. Defaults to `http://localhost:3000`.
  </Card>

  <Card title="CLAUDE_MODEL" icon="brain">
    Optional default Claude model on the server.
  </Card>

  <Card title="GEMINI_API_KEY" icon="globe">
    Optional. Only used for Browserbase Stagehand browser tools.
  </Card>
</Columns>

<Warning>
If `ANTHROPIC_API_KEY` is missing, server startup exits immediately. Old Vertex env vars are ignored.
</Warning>

## Verify the install

After installing, confirm the package imports correctly:

```bash
python -c "from aeon import Agent, Objective, Runtime; print('aeon-sdk ready')"
```

From this repository, maintainers can run the installed-SDK contract suite:

```bash
pnpm test:sdk
```

This creates a clean virtual environment, installs the local package non-editably with pip, and runs SDK contract tests against a local HTTP test server. No API credentials are required.

For a live smoke test against a running, configured server:

```bash
AEON_SERVER=http://localhost:3000 pnpm test:sdk:live
```
