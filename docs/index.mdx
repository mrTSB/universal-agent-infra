Aeon is a Python SDK and runtime for agents that need to pursue outcomes across days, months, or years without keeping a model process alive the whole time.

Define the agent, objective, tools, policy, memory, and budget in code. Aeon advances the work through bounded wake cycles, persists every ledger, and sleeps when there is nothing useful to do.

## Start in minutes

<Steps>
  <Step title="Run the Aeon server">
    ```bash
    pnpm install
    cp .env.example .env
    pnpm run agent
    ```
  </Step>

  <Step title="Install the Python SDK">
    ```bash
    python -m pip install aeon-sdk
    ```
  </Step>

  <Step title="Create a durable objective">
    ```python
    from aeon import Agent, Budget, Objective, Playbook, PlaybookStep, Policy

    agent = Agent(
        name="release-operator",
        models=["claude-sonnet-4-6"],
        system_prompt="Prefer reversible changes and verify every release.",
        policy=Policy(
            approval_required_tools=["deploy"],
            tool_risk_levels={"deploy": "critical"},
            workspace_only=True,
        ),
    )

    run = agent.pursue(Objective(
        goal="Keep this project releasable and publish verified releases.",
        success_criteria=["Tests pass", "Release evidence is recorded"],
        budget=Budget(max_cost_usd=20, max_cycles=10_000, max_turns_per_cycle=20),
        playbook=Playbook(
            name="verified-release",
            steps=[
                PlaybookStep("Inspect changes"),
                PlaybookStep("Run verification"),
                PlaybookStep("Publish with approval"),
            ],
        ),
    ))

    run.emit("repository.changed", {"ref": "main"}, dedupe_key="push-123")
    ```
  </Step>
</Steps>

<Tip>
  Waiting objectives are dormant. They consume no model tokens until an event, schedule, or explicit resume queues the next wake.
</Tip>

## Pick the right execution mode

<Tabs>
  <Tab title="Durable objective" icon="infinity">
    Use `pursue()` for work that needs durable state, external event wakeups, budgets, approvals, and long idle periods.

    ```python
    from aeon import Agent, Objective

    agent = Agent(name="operator")
    run = agent.pursue(Objective(goal="Keep the API healthy"))
    run.emit("health.changed", {"healthy": False}, dedupe_key="api-1042")
    ```
  </Tab>

  <Tab title="One-off task" icon="terminal">
    Use `run()` for short-lived interactive work that should complete in a single session.

    ```python
    from aeon import Agent, Task, SONNET

    agent = Agent(name="builder", models=[SONNET])
    run = agent.run(Task("Build a CLI todo app in Python"))
    print(run.result())
    ```
  </Tab>

  <Tab title="Swarm" icon="users">
    Use `Swarm` when an orchestrator should coordinate specialists that share a workspace and memory file.

    ```python
    from aeon import Swarm
    from aeon.swarm import SubAgent

    swarm = Swarm(agents=[
        SubAgent("planner", role="Break work into milestones"),
        SubAgent("critic", role="Review results and evidence"),
    ])

    run = swarm.run("Ship a tested CLI todo app")
    ```
  </Tab>
</Tabs>

## What Aeon persists

<Columns cols={3}>
  <Card title="Plan graph" icon="list-checks">
    Dependency-aware steps, attempts, evidence, and results for every wake.
  </Card>

  <Card title="Event journal" icon="radio">
    External and internal wake signals with deduplication and consumption state.
  </Card>

  <Card title="Action ledger" icon="scroll-text">
    Tool intent, risk, idempotency keys, attempts, outputs, and errors.
  </Card>

  <Card title="Memory" icon="brain">

  </Card>

  <Card title="Approvals" icon="badge-check">
